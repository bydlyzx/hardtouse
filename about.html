About Hard to Use

一个不是为了吐槽，
而是为了把“不好用”真正改成“好用”的长期项目。

这个网站在解决什么问题？

Hard to Use 关注的不是“用户为什么学不会”， 而是一个更根本的问题：

为什么这么多产品、软件和系统， 需要用户变聪明才能用？

当一个系统反复要求人记住规则、绕开陷阱、 为设计缺陷承担后果时， 问题并不在人，而在设计。

Hard to Use 的一个时间背景

在查看域名历史时，可以发现 hardtouse.com 曾在 2013 年被短暂使用过。

那一年，已经有人意识到： 很多软件并不是“学不会”， 而是“真的很难用”， 并试图用这个直接、不讨好的词来命名一个网站。

这个尝试并没有持续下去。 随后的十多年里，这个域名几乎完全沉寂， 像一个提前出现、却没有等到合适土壤的想法。

为什么这个问题在今天更严重了？

从 2013 年到现在，系统变得更复杂了：

  • 功能更多，流程更长
  • 系统之间高度耦合
  • 责任不断向使用者转移

与此同时，一个矛盾越来越明显： 技术在进步，但很多东西却越来越难用。

为什么是我在做这件事？

我长期站在“人和系统之间”解决问题： 修电脑、装系统、处理表格、梳理流程， 后来又在养老机构中， 面对大量与真实人群强相关的系统和制度。

这些问题并不“高端”， 但每天都在消耗人。 我逐渐意识到： 很多所谓的“麻烦”，并不是必要的。

这不是一个吐槽站

Hard to Use 并不满足于指出哪里不好用， 而是坚持做三件事：

  • 拆解：为什么这个设计必然导致出错
  • 判断：哪些负担本应由系统承担
  • 改进：给出可落地的优化方式

哪怕只是少一步操作、 少一次记忆、 少一个必然出错的点， 都是值得留下的改进。

一个迟到但并不晚的开始

从某种意义上说， Hard to Use 并不是“新建”的， 而是一次重新拾起。

拾起一个十多年前就已经出现， 却直到现在才真正拥有现实意义的判断。
